## 🔴 높음 (보안 / 안정성)

### [BUG] 스프레드시트 ID / GAS URL이 코드에 하드코딩되어 있음
- **파일**: `script.js` (L4–11), `GAS.js` (L19, L31–38), `update_data.yml` (L30–33)
- **문제**: 구글 시트 공개 URL, GAS 배포 URL, 스프레드시트 ID 등 민감 정보가 소스코드에 직접 노출됨
- **해결**: GitHub Secrets 또는 `.env` 파일로 분리하고, `.gitignore`에 추가
  ```
  SPREADSHEET_ID=xxxx
  GAS_URL=https://script.google.com/...
  ```

### [BUG] `no-cors` 모드 사용으로 GAS 응답 검증 불가
- **파일**: `script.js` (L470, L527)
- **문제**: `mode: 'no-cors'`로 POST 요청 시 응답 본문을 전혀 읽을 수 없어 성공/실패 구분 불가. 거래 기록 실패 시에도 사용자에게 "성공" 메시지가 출력됨
- **해결**: GAS 측에서 CORS 허용 헤더를 추가하고, `mode` 제거 후 실제 응답을 파싱하여 처리

### [BUG] 거래 제출 후 무조건 "성공" 메시지 출력
- **파일**: `script.js` (L533)
- **문제**: `no-cors` 모드이므로 실제 GAS 오류 여부와 무관하게 항상 ✅ 성공 표시
- **해결**: GAS CORS 해결 후, 응답 상태에 따른 분기 처리 필요

### [FEATURE] GAS에서 시장 지수 변화량(R열) 미입력
- **파일**: `GAS.js` (L191–198)
- **문제**: `updateMarketData()`에서 P열(현재가), Q열(변동률)만 입력. `script.js`에서는 R열(변화율, `row[17]`)도 사용하는데 GAS가 이를 채우지 않음
- **해결**: GAS에서 R열에도 수식 추가 또는 Q열 하나로 통합하여 혼선 제거

### [FEATURE] 자동 갱신 간격 고정값 (10분) — 사용자 설정 불가
- **파일**: `script.js` (L60–63)
- **해결**: UI에서 갱신 주기를 선택할 수 있는 드롭다운 추가 (5분 / 10분 / 30분 / 수동)

### [FEATURE] 거래 기록 입력 시 유효성 검사 부족
- **파일**: `script.js` (L497–502)
- **문제**: 수량/가격에 음수, 0, 비정상 값 입력 허용
- **해결**: 수량 > 0 조건, 가격 > 0 조건 추가 및 에러 메시지 출력

### [FEATURE] 모바일 화면에서 버블 차트 가독성 저하
- **파일**: `script.js` (L416), `style.css`

- **해결**: 화면 크기에 따라 버블 반경 스케일 동적 조정

### [FEATURE] 보유 종목 테이블 — 매수 평균가, 현재가 컬럼 없음
- **파일**: `index.html`, `script.js`
- **문제**: 사용자가 종목별 현재가/평균 매수가를 테이블에서 직접 확인 불가
- **해결**: Holdings 시트 데이터에서 해당 컬럼을 파싱하여 테이블에 추가 (토글로 숨기기 가능)

- **문제**: `:root` 변수가 라이트 모드 전용으로 고정됨
- **해결**: `@media (prefers-color-scheme: dark)` 또는 수동 토글 버튼 추가

### [UX] 새로고침(FAB) 버튼 클릭 시 로딩 인디케이터 없음
- **파일**: `script.js` (L65–68), `style.css`
- **문제**: 클릭 후 데이터가 로딩되는 동안 버튼 상태 변화 없음
- **해결**: 갱신 중 버튼에 스피너 또는 비활성화 처리 추가

### [UX] 차트 색상이 하드코딩되어 있음
- **파일**: `script.js` (L382, L402)
- **문제**: `rgba(54, 162, 235, ...)` 등 하드코딩된 색상값, CSS 변수와 일관성 없음
- **해결**: `getComputedStyle`로 CSS 변수를 읽거나 차트 색상 팔레트를 상수로 분리

### [UX] 오류 발생 시 사용자 메시지가 불명확함

- **문제**: `⚠️ ${task.key} 로드 실패 (URL/CORS 확인)` 형태로 영어 키 노출
- **해결**: 한국어 친화적 오류 메시지로 교체 (예: "요약 데이터 로드 실패")

### [CODE] GitHub Actions 워크플로우에 스크립트 인라인 작성
- **파일**: `.github/workflows/update_data.yml` (L24–89)

- **해결**: `scripts/update_snapshot.js` 파일로 분리하고 워크플로우에서 `node scripts/update_snapshot.js`로 호출

### [CODE] `package.json`에 실제 스크립트 명령 없음
